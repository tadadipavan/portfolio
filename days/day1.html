  2. Created an Amazon S3 Bucket:
    • Navigated to the S3 service in the AWS Management Console.
    • Created a new bucket named thadaristaticsiteshostingsings3 in the us-east-2 (Ohio) AWS Region.

    Successfully created bucket.jpg: Confirmation of S3 bucket creation.

    edit properties.jpg: S3 bucket properties tab.

  3. Configured S3 Bucket Public Access Settings & Policy:
    • Reviewed and adjusted "Block Public Access" settings, acknowledging the need to allow public access for static website hosting.
    • Accessed the "Permissions" tab of the created S3 bucket.
    • Edited the "Bucket policy" to allow s3:GetObject action for the * principal (anonymous users) on all objects (arn:aws:s3:::thadaristaticsiteshostingsings3/*). This makes the objects publicly readable.

    Block Public Access.jpg: Acknowledging changes for public access.

    policy.jpg: S3 bucket policy allowing public read access.

  4. Uploaded Website Content:
    • Navigated to the "Objects" tab of the S3 bucket.
    • Used the "Upload" feature to add index.html and error.html to the root of the bucket.

    html files uploaded.jpg: Confirmation of error.html and index.html being uploaded.

    index file link.jpg: Details for the index.html object in S3.

  5. Enabled S3 Static Website Hosting:
    • Went to the "Properties" tab of the S3 bucket.
    • Under "Static website hosting," clicked "Edit" and selected "Enable."
    • Specified index.html as the "Index document."
    • Specified error.html as the "Error document" (optional but good practice).
    • Saved the changes.

    enable static.jpg: Initial state where static website hosting is disabled (from properties tab).

    enab static.jpg: The "Enable" option selected for static website hosting (from edit view).

  6. Verified Website Accessibility:
    • Copied the "Bucket website endpoint" URL from the "Static website hosting" section in the S3 bucket properties.
    • Pasted the URL into a web browser and confirmed that index.html was displayed correctly.
    • Tested navigation to a non-existent path to verify error.html was served for 404 errors.
    Verification Note: While a screenshot of the live website isn't provided here, this step confirms successful access via the S3 endpoint.

Outcome: The static website is now live and directly accessible via the S3 provided endpoint. This completes the core S3 hosting component of Project 1.
